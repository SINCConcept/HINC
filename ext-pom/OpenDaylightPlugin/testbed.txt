Install Mininet and OpenDaylight controller:
 1 - Manual installation
 2 - Separate VM installation: http://www.brianlinkletter.com/using-the-opendaylight-sdn-controller-with-the-mininet-network-emulator/
 3 - Single VM installation: http://sdnhub.org/tutorials/opendaylight/


For case (2): 
1. Start OpenDaylight 
$ cd distribution-karaf-0.4.0-Beryllium/bin
$ ./karaf
opendaylight> feature:install odl-restconf odl-l2switch-switch odl-mdsal-apidocs odl-dlux-all

2. Start Mininet: 
sudo mn --topo linear,3 --mac --controller=remote,ip=100.96.9.107,port=6633 --switch ovs,protocols=OpenFlow13


3. Open Web page:
http://100.96.9.107:8181/index.html

4. create a OpenDaylight flow
curl -u admin:admin -H 'Content-Type: application/yang.data+xml' -X PUT -d @flow_data.xml http://100.96.9.107:8181/restconf/config/opendaylight-inventory:nodes/node/openflow:1/table/0/flow/10

5. Each flow in OpenDaylight consider a slice. With a set of devices, the controller can configure multiple flows with different parameters.

6. To connect other devices (e.g. a gateway) to the flow, we need to mount it to the controller: http://www.brocade.com/content/html/en/user-guide/SDN-Controller-2.1.0-User-Guide/GUID-6E3B7EFA-451B-4E12-93DC-CE7A6A946BD8.html


We have many other controller rather than OpenDaylight: http://archive.openflow.org/wk/index.php/OpenFlow_Tutorial
To install on Ubuntu, need to downgrade the kernel to 3.13
